Évaluation des spécifications générales - HTML

Spécification Analyse
Vos documents doivent débuter par le DOCTYPE HTML5
Le validateur W3C ne doit pas détecter d'erreurs sur vos 4 pages
doit être vérifié manuellement
L'élément BODY doit contenir comme enfants directs uniquement
des éléments avec une forte sémantique (HEADER,MAIN,FOOTER)
L'élément HEAD doit contenir l'élément TITLE et les META appropriées
Les éléments suivants doivent également être utilisés :
IMG, UL ou OL, A, P, H1, H2, DIV, SPAN
La structure du répertoire de travail doit être respectée
(dossier images / dossier css)
Toutes les images de votre site Web doivent être à l’intérieur
du dossier images (incluant celles de vos entités)
L'élément HEADER doit contenir un NAV avec des liens permettant
de naviguer vers"Accueil.html","Recherche.html"et"Favoris.html"
Le contenu des éléments HEADER et FOOTER doit être identique
sur vos 4 pages (uniquement le contenu de l'élément MAIN varie)
Utiliser au moins 2 nouvelles balises HTML (non vu dans le cours)
ET mettre en commentaire À CÔTÉ leurs utilités DANS VOS MOTS

Évaluation des spécifications générales - Bootstrap

Spécification Analyse
Inclure les bibliothèques nécessaires au bon fonctionnement de Bootstrap
Les enfants directs du body (header, main, footer) doivent être des conteneurs
La hiérarchie des éléments Bootstrap est respectée (container > row > col)
Les 2 types de conteneurs ont été utilisés au moins une fois (container, container-fluid)
Les 3 types de colonnes ont été utilisés au moins une fois (col, col-X, col-auto)
Des rangées imbriquées ont été utilisées au moins une fois
(mettre une rangée à l'intérieur d'une colonne)
Quelques images doivent être responsive (s'adapter à la grandeur du Viewport)
Lors d'un changement de taille de viewport entrainant une transition entre le mode "xs" et le mode "md" :
Certaines colonnes changent de configuration de taille
(en utilisant uniquement des classes Bootstrap)
Certains alignements de texte changent de configuration
(en utilisant uniquement des classes Bootstrap)
Certains éléments changent d'ordre d'affichage
(en utilisant uniquement des classes Bootstrap)
Certains éléments disparaissent ou apparaissent
(en utilisant uniquement des classes Bootstrap)

Spécifications propres à la page : Accueil.html

Spécification Analyse
Les informations des 3 entités parents sont présentées de manière élégante
Des statistiques basées sur les entités enfants des entités parents sont présentées
En mode "xs" et "sm" :
les entités parents doivent être présentées les uns en dessous des autres (à l’aide de Bootstrap)
En mode "md" et plus :
les entités parents doivent être présentés l'un à côté de l'autre (à l’aide de Bootstrap)
Cliquez sur les informations d’un entité parent te redirige vers la page « Recherche.html »

Spécifications propres à la page : Recherche.html

Spécification Analyse
Par rapport à la liste de résultat :
En mode "xs" et "sm" : les entités enfants doivent être présentés
les uns en dessous des autres (à l’aide de Bootstrap)
En mode "md" et plus : les entités enfants doivent être présentés
l’un à côté de l'autre (3 par lignes) (à l’aide de Bootstrap)
Les entités enfants doivent au minimum :
Posséder une bordure
Contenir l'image et le nom de l’entité enfant
Être cliquable (et apporter vers la page "Detail.html")
Par rapport au formulaire de recherche :
Il doit comporter 4 critères de recherche exactement
Il doit y avoir un filtre basé sur une propriété de type « texte »
en lien avec les entités enfants
Il doit y avoir un filtre basé sur une propriété de type « numérique »
en lien avec les entités enfants
Il doit y avoir une liste déroulante portant sur type « booléen »
en lien avec les entités enfants (Peu importe/Oui/Non)
Il doit y avoir des inputs de type « checkbok »
pour représenter les entités parents (coché par défaut)
Il doit avoir un bouton pour réinitialiser le formulaire (le bouton doit être fonctionnel)
Il doit avoir un bouton pour lancer la recherche (soumettre le formulaire)
Les données soumises par le formulaire doivent être envoyées par POST à l'adresse https://www.w3schools.com/action_page.php

Spécifications propres à la page : Detail.html

Spécification Analyse
Toutes les informations en lien avec l'entité enfant sont présentées à l'utilisateur (l'image de l'entité incluse)
Au minimum, le nom de l’entité parent doit être présentés à l’utilisateur
Un composant Bootstrap (autre qu'un simple bouton) est utilisé
(voir la section component : https://getbootstrap.com/docs/5.3/components/
Il doit y avoir un formulaire avec un bouton simple permettant l'envoi du formulaire
(pour ajouter l’entité à la liste des favoris) :
Un champ input hidden avec le name "nom" et le nom de votre entité enfant
Un bouton pour soumettre le formulaire avec comme
enfant l'icône à l'intérieur et un texte (ex : ajouté)
Les données soumis par le formulaire doivent être envoyées
par POST à l'adresse https://www.w3schools.com/action_page.php

Spécifications propres à la page : Favoris.html

Spécification Analyse
Le nombre d'éléments en favoris est affiché quelque part dans la page
Les éléments en favoris sont présentés sous forme de tableau :
Le tableau doit afficher les lignes impaires d'une couleur différente (avec l'aide de Bootstrap)
Le tableau est un tableau adaptatif (avec l'aide de Bootstrap)
Le tableau doit contenir au minimum les informations suivantes :
Le nom de l'entité parent
Le nom de l'entité enfant (dois être cliquable et apporter à la page detail.html)
L'image représentant l'entité enfant
Un formulaire par entité enfant avec un bouton simple permettant l'envoi du formulaire
(pour retirer l’entité de la liste des favoris)
Les formulaires doivent contenir uniquement deux éléments :
Un champ input hidden avec le name "nom" et le nom de votre entité enfant
Un bouton pour soumettre le formulaire
Les données soumis par le formulaire doivent être envoyées par POST à l'adresse https://www.w3schools.com/action_page.php
